# 常见问题

## 缓存命中率低怎么办？

精确缓存只有完全相同的请求才会命中。如果你的请求每次都有细微差异（比如时间戳、随机数等），命中率会很低。

商业版提供**语义缓存**，可以识别含义相同但表述不同的问题，命中率通常能提升 2-3 倍。

## 和上游厂商的缓存有什么区别？

上游厂商的 prompt cache 是**同一次对话内**的上下文缓存，换一个对话就没用了。

节能阀是**跨请求、跨用户**的全局缓存——只要有用户问过类似的问题，后面的人都能命中。

这完全是两个维度的东西，而且可以叠加使用。

## 会影响回复质量吗？

精确缓存返回的是完全相同的结果，质量 100% 一致。

如果你的场景需要每次输出都有变化（比如创意写作），可以关闭缓存或设置较短的 TTL。

## 支持流式输出吗？

支持。设置 `stream=true` 即可，和 OpenAI API 用法完全一致。

## 可以部署到生产环境吗？

开源版是单实例模式，适合中小团队和个人使用。

生产环境建议：
- 使用 Docker 部署
- 配置反向代理（Nginx）
- 设置 API Key 认证

对于大规模企业使用，建议使用商业版，支持高可用集群部署。
